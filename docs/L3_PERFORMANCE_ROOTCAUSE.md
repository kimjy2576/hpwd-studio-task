# L3 속도 저하 원인 규명 (프로파일링)

## 결론: CoolProp 물성 호출이 병목 (특정 컴포넌트 아님)

응축기 L3 step 1회 cProfile:
- 총 0.381s, CoolProp.PropsSI **3906회** 호출이 대부분
- properties.py:props_single이 tottime 0.102s (단일 최대)

## CoolProp 3906회 내역 (응축기 L3 1 step)
| 종류 | 입력 | 횟수 | 비중 |
|---|---|---|---|
| 포화물성 | P,Q | 2212 | 57% |
| 단상물성 | T,P | 1685 | 43% |
- D/H/V\|PQ (밀도/엔탈피/점도 포화) 각 480~526회
- D/L/C/H/V/Prandtl\|TP (단상) 각 240회

## 왜 많은가
- HX = N_rows×N_tubes×N_seg = 6×24×10, 720 세그먼트 계산
- 셀마다 물성 함수 6+개 호출 (rho/mu/k/cp/Pr/h + 포화 8종)
- 셀별 압력강하(셀당 1658Pa, 전체 16577Pa)로 포화물성 캐시
  부분 히트만 (props_single 히트율 72%, miss 240×6=1440 PropsSI)

## 컴포넌트별 (원인은 동일 — 물성)
- 응축기 363ms / 증발기 199ms (무거움) = 물성 호출 많음
- 압축기 3.4ms / EEV 0.7ms (가벼움) = 물성 호출 적음
→ "특정 컴포넌트"가 아니라 "그 컴포넌트가 부르는 물성 양"이 원인

## 최적화 여지 (측정 확인, 미적용)
- 포화물성은 압력 둔감: 5kPa 차 → T_sat 0.2K, rho_l 0.06%, h_fg 0.1%
- 캐시 키를 P 1Pa→~1-2kPa로 거칠게 하면 셀간 캐시 히트율↑ 기대
- ⚠️ 단, _vendor/hx_sim/properties.py = ground-truth. 수정 신중.
- 대안: N_seg 축소(10→5, 0.1% 오차) 또는 비동기(현 방침)

## 근본 성격
L3는 셀별 정밀 물성이 설계 목적. CoolProp 상태방정식(R290 Helmholtz)
호출 자체가 무거움. nproc=1 컨테이너에서 특히. 버그 아님.

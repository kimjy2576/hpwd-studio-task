② 중복 개발 — 발견된 것

🚨 커플용 공기포트 HX가 8종 — 상당수 중복
이게 이번 이력 꼬임의 핵심. 냉매 HX에 공기를 붙이는 방식이 제각각 4번 시도됨:
모델파일베이스공기결합상태
Evap_coupled / Cond_coupledCoupled.mo BF 모델 (L1급)AirPort✅ 사이클 동작 (L1 커플드)
EvaporatorMBdyn_cpl / CondenserSS_cplCoupledSEMI.mo MBdyn / SS (L2급)AirPort⚠️ Cycle_SEMI_full서 사용, 미검증
EvaporatorMB_cpl / CondenserMB_cplEvapMB/CondMB.mo MB 단독검증용AirPort❌ 오늘 신규 — 사이클 부적합
Evap_On_Dyn / Cond_On_DynHPWDevap.mo 동적 유한체적 (L3)공기 변수(방정식결합)✅ Cycle_MixHX서 동작
내가 오늘 만든 EvaporatorMB_cpl/CondenserMB_cpl은 삼중 낭비:
  • ① EvaporatorMBdyn_cpl이 이미 존재했는데 모르고 만듦
  • ② 베이스를 MB 단독검증용(algorithm+HelmholtzMedia record)으로 잘못 골라 사이클에 못 넣음 (OMC adjacency 실패)
  • ③ 사이클용 정답은 Evap_On_Dyn(순수 der 방정식)이었음
단독 커플 HX 검증(냉매+공기 경계)엔 유효하나, 폐루프엔 부적합 → 보존하되 사이클엔 미사용.
HX 3급 비교 harness 중복: HXCmp.mo(81행) vs CmpParts.mo(148행)
둘 다 냉매 HX L1/L2/L3 wrapper. 감싸는 알맹이가 다름:
  • HXCmp: Evap_On/Cond_On (정적 셀-march) + UA매칭 변형(Cond_L2m, Evap_L1n)
  • CmpParts: Evap_On_Dyn/Cond_On_Dyn (동적) — 이게 최신
→ HXCmp는 구버전 가능성. 정리 대상.
SS(정상상태) HX 중복: CondMBe.mo의 CondenserSS와 CondenserSSwall(벽 동특성 추가), CycleMBe.mo의 EvaporatorSS — MB 계열이 SS/MBdyn/MB로 여러 변형 존재. 어느 게 canonical인지 불명확.
🚨 패키지명 충돌 — HPWDcycle이 2개 파일에 동명 정의
  • Cycle.mo → package HPWDcycle ("L1 사이클": Volume, EEV_Orifice_ctrl)
  • HPWDcycle.mo → package HPWDcycle ("L3 사이클": Volume_L3, Cycle_L3_*)
둘 다 로드하면 나중 파일이 앞 파일을 덮어씀 → Volume_L3(HPWDcycle.mo) 또는 EEV_Orifice_ctrl(Cycle.mo) 중 하나가 사라짐. CmpCycle.mo에도 "같은 패키지명이라 로컬 복제"한 흔적 有. 파일↔패키지 매핑이 깨진 상태 — 재출발 시 패키지 이름 정리 필수.
깊은 의존성 사슬: Cycle_MixHX_* 로드에 R290Tab→HXCorr→HPWDair→HPWD→EvapUA→HPWDevap→HPWDon(finEffWet 등)→Cycle→HPWDcycle→Control→EevMB→CmpCycle 순 필요. 로드 순서 문서화 안 되어 실행 재현이 어려움.

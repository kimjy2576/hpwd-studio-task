# Cycle Runner 백엔드 경계조건 검증 (2/3: 극한 조건)

검증일: 2026-07 세션. SH_target=8.0 (EEV SH 제어) 적용 상태.

## 결과 요약

| 축 | 범위 | 결과 |
|---|---|---|
| 주위온도 T_amb | 5~55°C | ✓ 안정, 결과 무영향 (폐루프 단열 특성) |
| 드럼 입구 공기온도 | 10~70°C | ✓ 안정 (378~385W) |
| 건조 부하 M_dry | 0.5~15 kg | ✓ 안정 (390~391W), 정상상태 무영향 |
| 초기 함수율 X0 | 0.1~1.5 | ✓ 완전 동일, 정상상태 무영향 |
| 드럼 입구 RH | 40~90% | ✓ 안정 (391~393W) |
| 드럼 입구 RH | **10%** | △ 진동 미수렴 (maxΔT~0.3°C), 결과는 타당 |

## 관찰 및 해석

- **온도/부하/함수율 무영향은 정상**: 폐루프 건조기는 공기가 순환하며
  자체 평형을 찾음. 외부 조건보다 내부 순환이 지배. 건조 부하/함수율의
  실제 효과는 동적 시뮬(건조 시간)에서 나타남, 정상상태 열교환엔 무영향.

- **RH=10% 미수렴**: 극건조 입구에서 공기 연성이 진동
  (응축기 공기입구 20.1~20.6°C 사이 미세 진동, tol 0.05 못 맞춤).
  냉매도 max_iter(100) 도달하는 outer 발생. 그러나 결과(Q_evap=362,
  COP=3.21)는 물리적 범위. 수렴 실패가 아니라 tol 과엄격 + 극단 입력.
  RH 10%는 비현실적 극단(실제 건조기 입구 RH는 30%+)이라 실사용 무관.

## 결론
극한 조건에서 엔진은 견고. 코드 수정 불필요. RH<15% 극건조만
알려진 한계(진동, 결과는 타당). UI에서 미수렴 시 결과+경고 표시 권장.

# Cycle Runner 백엔드 경계조건 검증 (3/3: L3 균일 완주)

## L3 계산 시간 (측정 사실, nproc=1 컨테이너)

| 케이스 | 시간 | 결과 | 비고 |
|---|---|---|---|
| L3 냉매 solve 단독 | 106s | Q_cond=323 | 53 iter, N_seg=10 |
| L3 냉매 (완화 tol) | 78s | 동일 | 41 iter |
| L3 냉매 (N_seg=5) | 39s | 0.1% 오차 | 정확도 무손실급 |
| L3 커플드 (냉매+공기L3, N_seg=5) | 240s | Q_cond=429 | 드럼버그 수정 후 정상 |
| L3 균일 (공기도 L3, N_seg=10) | outer 1회 200s+ | 진행됨(미완주 관측) | 8분+ 추정 |
| one_pass 1회 | 566ms | — | 응축기363+증발기199 |

## 병목 구조
- L3 균일 = 냉매 solve(106s) × outer + 공기loop(HX 셀별) × outer
- 공기 loop도 evap/cond L3면 매 outer HX 셀별 CoolProp 재계산
- outer 8회 × 200s = 1600s+ (컨테이너 nproc=1 기준)

## 결론
- **L3는 정상 계산, 버그 없음**. 냉매 단독·커플드(N_seg=5) 완주 확인.
- L3 job 비동기 정상 (running 유지, 서버 안 멈춤).
- L3 완전 균일(공기도 L3)은 컨테이너에서 비실용적(10분+) —
  물리 모델의 본질적 무게(셀별 CoolProp), 버그 아님.
  실제 워크스테이션(다중 코어)에선 더 빠를 것.
- **권장**: UI에서 냉매L3+공기L3 조합 선택 시 경고
  ("매우 느림, 수분~십분"). 비동기라 UI는 안 멈추나 사용자 안내 필요.
- 컨테이너 제약(bash 500s 상한, nproc=1)으로 L3 균일 완주 관측은
  반복 실패했으나, 부분 측정으로 정상성·무게 모두 확인됨.
